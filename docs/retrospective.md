# Project Retrospective

## What Went Well

The strongest part of this project was the Services page. I’m especially proud of including proper image references so visitors can see exactly where the computer images came from. I felt that was important because it adds transparency and professionalism instead of just using random pictures. It makes the page feel more legitimate and intentional.

Using a shared external stylesheet also worked well. Once the CSS structure was stable, it helped keep everything visually consistent across the homepage, Services page, and Tools page. The gradient styling, layout structure, and card format all became easier to maintain once the foundation was correct.

## What Went Wrong

Integrating cards and images was more difficult than I initially expected. Getting spacing, alignment, and visuals correct required more adjustments than planned. At one point, the box shadows behind the cards were completely broken, which forced me to go back into the CSS and troubleshoot the issue. Fixing that helped me better understand how overflow, padding, and shadow properties interact.

I also struggled getting Visual Studio Code to sync properly with GitHub. Some commits did not behave as expected, which slowed progress and created unnecessary frustration. That technical friction made small updates take longer than they should have.

## What I Would Do Differently

If I were to redo this project, I would finalize the layout structure and navigation before focusing on visual styling. I would also test GitHub integration early with small commits to make sure the workflow is stable from the beginning.

Overall, this project strengthened my understanding of HTML structure, CSS layout behavior, and version control. It reinforced the importance of planning, testing early, and controlling scope to avoid unnecessary complications.
