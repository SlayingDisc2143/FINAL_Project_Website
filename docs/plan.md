# Project Plan

## Project Overview

This project involves designing, structuring, and publishing a professional computer maintenance website using HTML and CSS. The site includes a homepage (which serves as the About page), a Services page, and a Tools page.

The focus is clarity, professionalism, and technical consistency.

## Risk Management (TAME Framework)

## Risk 1: Scope Creep

**Trigger:** Adding unnecessary features, extra pages, or complex layouts beyond requirements.

- **Avoid:** Follow approved scope and locked page list.
- **Transfer:** N/A
- **Accept:** Minor aesthetic upgrades if time allows.
- **Mitigate:** Complete core structure before adding enhancements.
- **Eliminate:** Freeze feature additions after navigation and styling are finalized.

## Risk 2: Broken Links

**Trigger:** Incorrect file names or navigation paths.

- **Avoid:** Maintain consistent naming conventions.
- **Transfer:** N/A
- **Accept:** Minor formatting errors that can be corrected quickly.
- **Mitigate:** Perform full manual click-test before submission.
- **Eliminate:** Keep folder structure simple (root HTML + single CSS).

## Risk 3: Inconsistent Design

**Trigger:** Styling pages separately instead of using shared CSS.

- **Avoid:** Use a single external stylesheet.
- **Transfer:** N/A
- **Accept:** Small browser differences.
- **Mitigate:** Review layout on both desktop and mobile.
- **Eliminate:** Reuse structural HTML patterns across pages.

## Risk 4: Time Constraints

**Trigger:** Balancing coursework and responsibilities.

- **Avoid:** Build a complete basic version early.
- **Transfer:** N/A
- **Accept:** Site remains visually simple.
- **Mitigate:** Work page-by-page in focused sessions.
- **Eliminate:** Prioritize required deliverables first.

## Work Breakdown Structure

1. Define project scope
2. Identify required pages
3. Write project plan
4. Outline navigation structure
5. Build homepage (About)
6. Build Services page
7. Build Tools page
8. Create shared stylesheet
9. Implement consistent navigation
10. Apply visual theme and gradients
11. Add responsive layout adjustments
12. Perform link testing
13. Finalize scope document
14. Write retrospective
15. Update README

## Definition of Completion

The project is complete when:

- All pages load correctly
- Navigation works on every page
- Styling is consistent
- Services are clearly described
- Tools are clearly explained
- Documentation folder is complete
- GitHub Pages is live
